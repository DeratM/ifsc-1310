Thoughts on Web Accessibility

My thoughts on web accessibility are that they are one of the most important things a website can have. Web accessibility for me means a website that has multiple ways to be used for people with different kinds of disabilities, for example, people with dexterity problems or eyesight issues to name a couple. The reason why this is so important is that these accessibility tools help users navigate and interact with websites that otherwise they wouldn't be able to. So I feel like accessibility for anyone with any kind of disability is a basic human right. Especially as web developers, we all have a duty to keep all types of users in mind, not only the able-bodied ones. Moreover, there are benefits to having a more accessible website. For example, all types of users will find it easier to use this website so they'll come back, in turn, this will cause traction to the website and raise its SEO, so more people will come and use it.

Progressive Enhancement Writing Assignment

After reading about Progressive Enhancement and Graceful Degradation it seems like such an obvious issue yet it isn't. First, let me explain what is graceful degradation because it's somewhat the simplest one out of the two. Graceful degradation is the idea that web devs have a bias towards more advanced browsers. This bias causes these devs to put more time and effort into their websites for these “advanced” browsers and then leave the rest of the “crappy” browsers out or just make a worse version of the website. This bias is extremely unfair and just a crappy move when creating a website that is intended to hit a large audience. That is where the idea of progressive enhancement comes in. It's an idea presented by Steve Champion and Nick Finick. This idea explains that web devs should focus on content before anything so all users gain the same experience and information that is needed. I quite enjoy the example of the peanut M&M, the content is the peanut, the chocolate is the presentation, and the client-side scripting. Overall progressive enhancement is such an important thing in web development because as web developers we shouldn't have a bias towards one group of users. As web devs, we should focus on trying to make the best and easiest accessible websites, and with graceful degradation that isn't possible. Finally, this article by W3 (Graceful degradation versus progressive enhancement) explains great use cases where both Progressive Enhancement and Graceful Degradation are needed.
